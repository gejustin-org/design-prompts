Talk 1

The Great Platform Convergence

Kelsey Hightower · Google Cloud · 45 minutes

TL;DR
Platform engineering has evolved from "Kubernetes + YAML" to AI-native ecosystems that understand developer intent and automate the entire software delivery pipeline.

Core Insights

  1. Intent-based platforms are replacing configuration-heavy systems. Developers describe what they want ("deploy a resilient API"), not how to do it.
  2. AI code assistants are now platform citizens — they understand your infrastructure, deployment patterns, and team conventions.
  3. The platform IS the documentation. Self-describing systems that teach themselves to new team members through interactive onboarding.
  4. GitOps evolved into "ConversationOps" where platform changes happen through natural language conversations with AI agents.
The best platform is the one developers forget exists — until they need it to do something impossible.
— Kelsey Hightower

Talk 2

AI-Native Development Workflows

Sarah Chen · Anthropic · 35 minutes

TL;DR
The most successful teams in 2026 treat AI as infrastructure, not a tool — embedded throughout the platform to provide continuous intelligence about code quality, security, and performance.

The New Development Loop

  1. Context-aware code generation that understands your team's patterns, naming conventions, and architectural decisions.
  2. Predictive testing where AI suggests test cases based on code changes and historical bug patterns.
  3. Intelligent deployment strategies that adjust rollout speed based on risk assessment and system health.
  4. Automated incident response with AI agents that can diagnose, mitigate, and learn from production issues.
We're not replacing developers — we're giving them superpowers. AI handles the routine; humans handle the creative.
— Sarah Chen

Platform Engineering Implications

  • Infrastructure as intelligence — platforms need AI-native architectures from day one
  • Trust and transparency become critical as AI makes more autonomous decisions
  • Human-in-the-loop workflows for high-risk changes and architectural decisions
Talk 3

The Developer Experience Revolution

Alex Rivera · Vercel · 40 minutes

TL;DR
Developer experience is now the primary competitive advantage. The best platforms eliminate cognitive load through intelligent defaults, predictive workflows, and contextual guidance.

DX Principles for 2026

  1. Zero-config by default, infinitely configurable — platforms should work immediately but allow deep customization when needed.
  2. Contextual intelligence — the platform knows what you're working on and suggests relevant tools, documentation, and next steps.
  3. Observability as a first-class experience — monitoring and debugging are as smooth as writing code.
  4. Cross-team collaboration built in — platforms connect developers with design, product, and operations seamlessly.

Theme 1: Automation First

Theme 1A

Intelligent Pipeline Evolution

  • CI/CD pipelines that adapt based on code changes and risk assessment
  • Automated rollback decisions driven by real-time health metrics
  • Self-healing infrastructure that resolves common issues autonomously
  • Dynamic resource allocation based on predicted demand patterns
Implication: Platform teams shift from building pipelines to training intelligent systems that understand deployment patterns and business context.
Theme 1B

Policy-as-Code Renaissance

  • Security policies that evolve with threat landscape
  • Compliance frameworks that adapt to regulatory changes
  • Cost optimization rules that learn from usage patterns
  • Performance budgets that adjust based on user experience data
Implication: Traditional "policy enforcement" becomes "policy learning" — systems that understand intent and adapt rules intelligently.

Theme 2: Embedded Intelligence

Theme 2A

Code Intelligence at Platform Scale

  • Cross-repository understanding of architectural patterns
  • Automated dependency management with security and compatibility scoring
  • Intelligent code review that understands business logic
  • Predictive bug detection based on team coding patterns
Implication: The platform becomes a participant in code quality discussions, not just an enforcement mechanism.
Theme 2B

Operational Intelligence

  • Proactive incident prediction based on system behavior patterns
  • Automated root cause analysis with natural language explanations
  • Capacity planning driven by ML models of usage patterns
  • Security threat detection that understands application context
Implication: Operations shift from reactive firefighting to proactive orchestration of intelligent systems.

Theme 3: Experience Design

Theme 3A

Contextual Developer Interfaces

  • Dashboards that adapt to current development phase
  • Documentation that updates based on code changes
  • Workflow suggestions based on team productivity patterns
  • Personalized learning paths for new platform features
Implication: Platform UX becomes as important as API design — interfaces that learn and adapt to individual developer preferences.

Immediate Action Items

  1. 1
    Audit current toolchain complexity — Map every tool developers use and identify consolidation opportunities. Target: 50% reduction in context switches.
  2. 2
    Pilot AI-assisted code review — Start with automated style and security checks, gradually expand to architectural feedback.
  3. 3
    Implement intent-based deployment — Begin with simple declarative configs that hide complex orchestration details.
  4. 4
    Build platform intelligence baseline — Start collecting workflow data to train future AI assistants on team patterns.
  5. 5
    Establish DX metrics — Measure developer satisfaction, onboarding time, and cognitive load to track platform evolution.

Meta-Insights: What This Means for Platform Engineering

  1. Platforms are becoming products — with real users, user research, and product management disciplines.
  2. AI isn't a feature, it's infrastructure — the most successful platforms embed intelligence at every layer.
  3. Developer experience beats technical sophistication — simple, intuitive platforms win over feature-rich complex ones.
  4. The platform team is becoming a design team — focused on experience design, not just technical architecture.
  5. Intent-based systems are the new API — developers describe goals, not implementation steps.

Strategic Implications

  • Hiring priorities shift — need more UX designers, product managers, and AI/ML engineers on platform teams
  • Budget allocation changes — more investment in developer tooling, less in traditional infrastructure
  • Success metrics evolve — from uptime and performance to developer satisfaction and productivity
  • Platform architecture decisions — choose technologies that enable AI integration and contextual intelligence
